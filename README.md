# AI-Driven Cloud Resource Allocation

AI/ML-based forecasting for **cloud resource usage** (CPU & Memory) using time-series feature engineering and multi-output regression models.  
This project trains and evaluates **XGBoost** and **Random Forest** models to predict near-future resource consumption from workload, storage usage, and engineered time/lag features—supporting smarter, proactive cloud resource allocation decisions.

## Overview

Modern cloud environments need **predictive scaling and allocation** to reduce:
- over-provisioning cost (wasted capacity),
- under-provisioning risk (SLA violations),
- and reactive scaling delays.

This repository demonstrates a practical ML workflow that:
1. loads a time-stamped dataset of system metrics,
2. creates time-based and lag-based features,
3. trains a multi-output regressor to predict **CPU utilization** and **memory usage**,
4. evaluates and visualizes predictions.

---

## Project Goals

- Predict **CPU utilization** and **memory usage** using ML regression.
- Compare performance between:
  - **XGBoost** multi-output regression
  - **Random Forest** multi-output regression
- Provide a simple pipeline to:
  - train models,
  - test models,
  - export prediction results,
  - plot actual vs predicted usage.

---

## Approach

### Feature Engineering
The training/testing scripts implement:
- **Time features** from timestamp:
  - hour, day_of_week, week_of_year, day_of_month, month, is_weekend
- **Lag features** for short-term history:
  - cpu_lag_{1,2,3,6,12}
  - memory_lag_{1,2,3,6,12}

### Modeling
Both model families are wrapped using:
- `sklearn.multioutput.MultiOutputRegressor`

Targets predicted:
- `cpu_utilization`
- `memory_usage`

## Repository Structure

Key files in the repo:

- `dataset.csv`  
  Main dataset used for training/testing.

- **XGBoost pipeline**
  - `xg_train.py` — trains XGBoost model and saves `final_xgboost_model.pkl`
  - `xg_test.py` — evaluates model, prints metrics, and writes `prediction_results.csv`

- **Random Forest pipeline**
  - `ran_train.py` — trains RandomForest model and saves `rf_cpu_memory_model.pkl`
  - `ran_test.py` — loads RF model and displays plots (actual vs predicted + error plots)

- `plots.py`  
  Plots CPU & Memory predictions from `prediction_results.csv`.

- `prediction_results.csv`  
  Output file generated by `xg_test.py` (actual vs predicted).

- Images
  - `CPU UTilization.png`
  - `Memory usage prediction.png`

## Dataset

The scripts expect `dataset.csv` in the repository root.

Expected columns (used in the code):
- `timestamp`
- `cpu_utilization`
- `memory_usage`
- `storage_usage`
- `workload`
- `Resource Allocation` (present in dataset; scripts convert to `Resource_Allocation` after cleaning)

> The scripts clean column names by stripping spaces and replacing `" "` with `"_"`.

## Installation

### Prerequisites
- Python 3.9+ recommended

### Create and activate a virtual environment (recommended)

**Windows (PowerShell)**
```bash
python -m venv .venv
.venv\Scripts\Activate.ps1
```

**macOS/Linux**
```bash
python -m venv .venv
source .venv/bin/activate
```

### Install dependencies

This repo doesn’t currently include a `requirements.txt`, so install the required packages manually:

```bash
pip install pandas numpy scikit-learn xgboost joblib matplotlib
```

(Optional but helpful)
```bash
pip install seaborn
```

## How to Run

All scripts assume they are run from the repository root (where `dataset.csv` is located).

### 1) Train Models

#### Train XGBoost
```bash
python xg_train.py
```

This creates:
- `final_xgboost_model.pkl`

#### Train Random Forest
```bash
python ran_train.py
```

This creates:
- `rf_cpu_memory_model.pkl`

---

### 2) Evaluate / Test Models

#### Test XGBoost + export results
```bash
python xg_test.py
```

This prints metrics (MAE/MSE/RMSE/R²) and generates:
- `prediction_results.csv`

#### Test Random Forest (plots + errors)
```bash
python ran_test.py
```

This shows:
- CPU actual vs predicted
- Memory actual vs predicted
- CPU prediction error plot
- Memory prediction error plot

---

### 3) Plot Predictions

After generating `prediction_results.csv` (typically via `xg_test.py`):

```bash
python plots.py
```

This displays:
- CPU Utilization Prediction (Actual vs Predicted)
- Memory Usage Prediction (Actual vs Predicted)

## Notes / Troubleshooting

- **File naming**: Your image name includes a space and a capital “T” in `CPU UTilization.png`. Keep the exact name if you reference it.
- **Matplotlib windows not showing**:
  - Ensure you’re not running in a headless environment, or use a notebook.
- **`ModuleNotFoundError`**:
  - Install dependencies with the `pip install ...` command above.
- **Dataset column mismatches**:
  - The scripts automatically normalize column names by stripping spaces and replacing them with underscores.
